About me

I'm Knot (usually appended with the number 126)! I am some mix of a programmer, engineer, mathematican, artist and idiot and I can't describe myself in any way other than that.

The story

I believe I first started programming in Python when I was 13 by reading the introduction to programming that comes with the python documentation. I started programming in C shortly after, since I liked it's simplicity.

I made a few dumb things that didn't do anything and later imagined Quick Run, an endless running game. I planned to write it in Python, but of course that's a terrible idea, so I started using C++.

20 pretty meaningless git commits later, I decided that C++ is actually a really terrible language, and starting writing in C instead in 2021. Using that, I made the first version of the Trestle engine — which I have now dubbed the Quick Run engine — which actually did manage to do something. It had basic physics and graphics and a Lua based scripting system and could be identified as being a game.

Of course, I was a bit hyperfocused on performance, and forgot to actually make the game, and so the entire project stopped any major development in late 2021 and completely died in September 2022.

But wait! It turns out I was doing a lot of other crap too. Shatter — at the time called Smash Hit Blender Tools or some variant of that — was first released in September 2020 as an extension for Blender to create parts of levels called segments for the mobile game Smash Hit.

On 17 Feburary 2021, I did what I still consider to be one of the Top Ten Things I Was Happiest With Myself Doing which was figuring out the pre-baked lighting/mesh file format for Smash Hit.

In April 2021, I tried to kill Shatter with Carambola, an editor made in Godot that, importantly, showed the textures of boxes as they appeared in the scene. As it turns out Shatter had the weapon of being easy to develop and easy to polish and so Carambola also died in 2021.

Also sometime in 2021, I saw the game Croc: Legend of the Gobbos — which you probably know as the Scrimblo Bimblo game — and felt the main character was so cute that I started reverse engineering the custom scripting language (or as they called it a "strategy language") that the game uses as the Stratigise project. I'm still working on that project occasionally and recently (in 2023) made more progress in modding the game.

In 2022, I mostly worked on Shatter. I fixed the mesh baker (which was really buggy and slow at the time) and made various tweaks and quality of life improvements.

I didn't do much during 2022 (or at least I cannot remember anything), though I did really start to think about a Teardown-like Tails the Fox fangame I wanted to make. I still think about it today, admittedly. I'll write about my dreams for it soon...

At the start of 2023, I worked on the Smash Hit Lab community website constantly for about four months. I really did enjoy working on it, though using PHP was a bit of a pain and the website was still very much Web 2.0 1.0. It was of course shut down in 2023, but more because I left the Smash Hit Lab staff team after some disagreement.

Then soon after the SHL website shutdown, I was trying to dump some beta versions of Smash Hit — because I fucked it up the first time and also I'm now a game preservationist I guess — and I noticed Touch Pets: Cats on an old iPhone, a game from 2010 that had it's servers shut down in 2013 and was online only and therefore unplayable ... of course, I had to fix that and so went into the insanity of reverse engineering Objective C. I don't think I've recovered from that, but I got the game to boot at least.

Starting in August 2023, I started work on Autogen for Shatter — a set of in-editor procedureal generation tools I had wanted to work on for a long time. This allowed significantly speeding up the creation of detailed stages while still allowing creators to tweak the results.

I mention that specifically because now I'm thinking that some previous dreams of a procedureal content-driven engine could not only be possible but also actually viable, if done well.

And so as an extercise to actually start seriously programming again, I have been developing the Mot 2D engine for a game I'm calling Knock. It's basically just a physics game, but the engine will be a "0.1 proof of concept" for a PCG engine. I'm not sure if Mot will actually last, since I might just start to work on Trestle again, but I can at least try not to kill it. :)
