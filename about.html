The graphics pipeline in GPU compute

A completely programmable graphics pipeline is a very, very intresting idea. Even if having RT cores or the traditional rasterisation pipeline is basically guaranteed to be more efficient, I think the programmability will make up for itself.

High level graphics primitives

Instead of just rendering triangles, why not render smooth surfaces, like Bezier patches? NURBS surfaces? Entire implicit equations?

Procedural content generation

Using code as the method of input to an asset, like a texture, model, music, level layout, font or anything else, is a really good idea. It essentially removes the need for a separate preview process as things that be seen directly in engine.
