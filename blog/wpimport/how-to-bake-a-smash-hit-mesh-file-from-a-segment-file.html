How to Bake a Smash Hit Mesh File from a Segment File

Note: I am aware of how full of it this post is now. I will leave it up for reference and as an example of why I shouldn’t make posts just because I can…

It’s only been a day since it has been possible to easily export mesh files, and there are already many people who want to take advantage of it. So, let’s see how to make a mesh file from a segment’s XML file!

Blender Tools

Firstly, if you use Smash Hit Blender Tools, download the newest version and install it. It will have support for exporting meshes alongside your segment XML files, so you don’t have to manually do it every time. It’s based on the same exporter function from MeshBake, so it is the same quality of export and much more convenient. All you need to do is make sure that Box Export Mode is set to Mesh Bake, which it will be by default on all files. Then, you can export your segments with mesh files alongside them.

MeshBake

However, if you use an editor that doesn’t support baking mesh files, or you edit your mesh files by hand, you will need to use a small utility called MeshBake. Download the latest version of Mediocre Tools pack and unzip it, then install python.

Next, open the folder where you extracted Mediocre Tools pack and go into the folder called “meshbake.” On Windows, hold SHIFT and right click, and click “Open in Command Prompt/Powershell.” On a Linux based system, use the normal method for opening a terminal emulator in the current directory.

When you are at this window, type the following:

python ./meshbake.py PathToXMLFile PathWhereMeshFileShouldGo

If its says something about python not being found, try this:

python3 ./meshbake.py PathToXMLFile PathWhereMeshFileShouldGo

Make sure to replace PathToXMLFile with the path to your segment’s XML file. You should just be able to drag and drop the XML file into the window. Also, make sure to set PathWhereMeshFileShouldGo to the name of the file you want your mesh to be exported to. After you have typed all of this, run it by pressing enter.

Here are a few examples if you are confused:

# On windows:
> python ./meshbake.py "C:\Users\SomeName\Documents\Smash Hit Segments\somesegment.xml.mp3" "C:\Users\SomeName\Documents\Smash Hit Segments\somesegment.mesh.mp3"

# On Linux:
$ python ./meshbake.py /home/somename/Documents/SmashHitSegments/somesegment.xml.mp3 /home/somename/Doucments/SmashHitSegments/somesegment.mesh.mp3

Please note that MeshBake does not support mesh files that have been compressed (that is, the ones that end in .xml.gz.mp3).

If everything went well, you should now have a mesh file alongside your XML file. You should be able to copy these files into the APK, load them and then play though them and see that you have real meshes instead of the stonehack obstacles.

Notes

  • You may notice that the textures don’t tile correctly. This is a fault with the exporter, but it is still better than using stonehack obstacles. (This also means that the tile size properties will be ignored.)
  • If you were using stonehack obstacles and have switched to using baked meshes, make sure that all of your visibility properties are set correctly. If they are not, the boxes may fail to show up correctly and use fallback values. Only boxes which have their visible property set to “1” (or checked in blender tools) will be exported to the mesh.
  • Note that is is not possible to use templates at the moment, and it will probably never be possible to use templates with Blender Tools.
  • The exported mesh will likely be somewhat inefficient because it does not take advantage of Index/Element Buffer Objects, which the mesh files often use to improve performance in the game.
